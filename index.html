Welcome!

This is the home of H#: An open source, functional, math-oriented scripting language. It was made with readability and friendliness to math enthusiasts in mind, and aims to be strongest when trying to make short programs with self-explanatory code.

This page is to share the strengths, weaknesses, documentation, and tutorials on H#.




Concise, Weakly-Typed, and Purely Functional

let primes = filterprime | [2..500]     where filterprime|[] = [], filterprime =         x -> head | x : filterprime | (filter | (l -> l % (head | x) /= 0) | ( tail | x)) //Generates all prime numbers from 2 to 500;

H# is purely functional, meaning it enforces immutable values, and no side-effect-producing functions. Referential transparency was essential when creating H#, but so was friendliness to the user. Operations like getstr and getnum will return a valid string, unlike Haskell which returns IO String from input functions. However, any IO operation requires a world parameter, so the code stays referentially transparent. H# has a nice blend of lazy evaluation and eager evaluation, which aims to find a happy medium of both types of evaluation systems. As far as typing goes, H# is almost non-typed. It's run via an internal CAS, which allows crazy operations like 3+4+x where x isn't defined to just simplify into 7+x. As a bonus, mismatched types just don't evaluate, so 4+"hi" is a legal return value from a function, or as output onto the console screen.
